# 丽水市非物质文化遗产平台 — 答辩用：项目整体介绍与后端流程

本文档面向答辩准备，从零介绍：项目是什么、技术栈、目录结构、Spring Boot 后端请求流程、前后端如何对接，以及一条完整请求从点击到数据库的链路。按顺序看完即可对项目有整体认识，便于口头介绍和回答老师提问。

## 一、项目是什么（一句话 + 扩展）

**一句话**：这是一个基于 **Vue.js 前端 + Spring Boot 后端 + MySQL 数据库** 的丽水市非物质文化遗产展示与管理平台，分为**用户端**和**管理员端**，实现非遗项目、资讯、传承人、传承故事、活动的展示与后台管理，以及基于浏览/收藏的简单推荐算法。

**扩展**：
- **用户端**：浏览非遗项目/资讯/传承人/传承故事、活动预约、收藏项目、AI 咨询、个人中心等。
- **管理员端**：对非遗项目、资讯、传承人、传承故事、活动、用户进行增删改查；推荐权重配置；数据统计与可视化。
- **推荐**：根据用户浏览和收藏记录，按非遗类型加权打分，从得分高的类型中推荐项目，管理员可配置各类型的基础权重、浏览权重、收藏权重。

## 二、技术栈（答辩时按层次说）

| 层次     | 技术           | 作用简述 |
|----------|----------------|----------|
| 前端     | Vue 3          | 页面与交互，组件化开发 |
| 前端     | Vue Router     | 单页应用路由（如 /projects、/admin） |

| 后端     | Spring Security| 登录认证、权限控制（JWT 无状态） |
| 后端     | Spring Data JPA| 对象关系映射，操作数据库 |

论文里写的是 MyBatis，实际项目用的是 **JPA**（Spring Data JPA），答辩时以实际为准：持久层是 JPA + MySQL。

## 四、Spring Boot 后端流程（请求怎么走一遍）

可以记一句口诀：**请求先进 Security，再进 Controller，Controller 调 Service，Service 调 Repository 查数据库，最后把结果封装成 JSON 返回。**

### 4.1 整体顺序（按时间顺序）

1. **浏览器发请求**  
   例如：`GET http://localhost:5173/api/projects?page=0&size=10`  
   前端开发时，Vite 会把 `/api` 代理到 `http://localhost:8080`，所以实际到达后端的是：`GET http://localhost:8080/api/projects?page=0&size=10`。

2. **Spring Security 过滤**  
   - 所有 `/api/**` 请求会先经过 `JwtAuthFilter`。  
   - 从请求头里取 `Authorization: Bearer <token>`，若有且有效，就把当前用户 id、角色放进 Security 上下文；没有或无效就不放，后续接口若要求“已登录”则会返回 401。  
   - 配置里：`/api/auth/login`、`/api/auth/register`、`/api/auth/admin/login` 以及部分 GET（如 `/api/projects/**`、`/api/news/**` 等）是**放行**的；`/api/admin/**` 要求 **ADMIN** 角色；其余 `/api/**` 要求**已登录**。

3. **进入 Controller**  
   - 请求路径和 Method 匹配到某个 `@RestController` 的某个方法。  
   - 例如：`GET /api/projects` → `IchProjectController.list(...)`。  
   - 参数从 URL 查询串、路径、Body 里解析出来（如 `keyword`、`page`、`size`）。

4. **Controller 调用 Service**  
   - Controller 只做“接收参数、调 Service、封装返回”。  
   - 例如：`IchProjectController.list()` 里调用 `projectService.list(keyword, region, heritageLevel, ichType, page, size)`，得到分页结果后，用 `ResponseEntity.ok(Map.of("success", true, "data", ...))` 返回 JSON。

5. **Service 调用 Repository**  
   - Service 里写业务逻辑（如组合条件、调用推荐算法等）。  
   - 例如：`IchProjectService.list()` 里调用 `projectRepository.searchByConditions(...)`，返回的是 JPA 的 `Page<IchProject>`。

6. **Repository 访问数据库**  
   - Repository 是 JPA 接口，实现由 Spring 自动生成。  
   - 方法名或 `@Query` 会转成 SQL，对 MySQL 进行查询，结果映射成实体对象（如 `IchProject`）返回给 Service。

7. **原路返回**  
   - Service 把结果给 Controller，Controller 封装成统一格式（如 `{ success: true, data: [...] }`）通过 HTTP 响应返回给浏览器；前端 axios 收到 JSON，再渲染页面。

### 4.2 用一张图串起来（答辩时可画）

```
浏览器 → [Vite 代理] → 8080 端口
         ↓
    JwtAuthFilter（看 Token，设用户）
         ↓
    Controller（接请求、调 Service）
         ↓
    Service（业务逻辑、调 Repository）
         ↓
    Repository（JPA → SQL → MySQL）
         ↓
    Service ← Repository 结果
         ↓
    Controller 封装 JSON
         ↓
    浏览器 ← 响应
```

## 五、前后端是怎么连起来的（重要）

分三块：**开发时的请求地址、前端如何发请求、后端如何允许前端访问**。

### 5.1 开发时：前端为什么能访问到后端？

- 前端运行在 **http://localhost:5173**（Vite 默认）。  
- 后端运行在 **http://localhost:8080**。  
- 浏览器里页面是 5173，若直接写 `axios.get('http://localhost:8080/api/projects')` 会跨域；所以前端**不写死 8080**，而是写相对路径 `/api/projects`。  
- 在 **frontend/vite.config.js** 里配置了开发服务器代理：
  - 请求路径以 `/api`、`/pictures`、`/uploads` 开头时，Vite 会把这些请求**转发到 8080**。  
- 这样浏览器看到的是“同源”（都是 5173），实际数据来自 8080，没有跨域问题。

### 5.2 前端怎么发请求、带登录态？

- 所有请求都通过 **frontend/src/utils/request.js** 里的 **axios 实例** 发出。  
- 在**请求拦截器**里：从 `localStorage` 取 `token`，若有，就在请求头里加上 `Authorization: Bearer <token>`。  
- 在**响应拦截器**里：若返回 401，就清掉本地 token 并跳转到登录页（用户端 / 管理员端分别跳不同登录页）。  
- 具体接口不写 axios，而是写在 **frontend/src/api/index.js** 里，例如：
  - `getProjectList(params)` → `request.get('/api/projects', { params })`
  - `getRecommendProjects()` → `request.get('/api/projects/recommend')`
  - `adminLogin(data)` → `request.post('/api/auth/admin/login', data)`  
- 页面里（如 `Projects.vue`）只要 `import { getProjectList } from '../api'`，然后 `await getProjectList({ page: 0, size: 12 })` 即可，既简洁又统一带 Token。

### 5.3 后端怎么允许前端访问？

- 在 **WebMvcConfig** 里配置了 **CORS**：  
  - 对路径 `/api/**` 允许任意来源（`allowedOriginPatterns("*")`）、常用方法（GET、POST、PUT、DELETE、OPTIONS）、任意请求头、携带凭证。  
- 这样 5173 的页面请求 8080 的接口时，浏览器不会因同源策略拦掉。

**总结**：  
- 开发时：Vite 代理把 `/api` 转到 8080；  
- 前端用 axios 封装 + api/index.js 发请求，并自动带 Token；  
- 后端用 CORS 允许前端域名访问。  
三者配合，前后端就“连起来”了。

## 六、一条完整例子：从“非遗项目列表”点击到数据库

方便答辩时讲“一个功能从头到尾是怎么实现的”。

1. **用户操作**  
   打开“非遗项目”页（例如路由 `/projects`，对应 `Projects.vue`）。

2. **页面加载时**  
   - `Projects.vue` 的 `onMounted` 里调用 `load(0)`。  
   - `load()` 里调用 `getProjectList({ keyword, region, heritageLevel, ichType, page: 0, size: 12 })`。

3. **前端 API 层**  
   - `api/index.js` 里 `getProjectList(params)` 执行 `request.get('/api/projects', { params })`。  
   - `request` 在请求头里加上 `Authorization: Bearer <token>`（若已登录）。

4. **请求到达后端**  
   - 若开发环境，请求先被 Vite 代理到 `http://localhost:8080/api/projects?page=0&size=12&...`。  
   - 进入 Spring：先经 `JwtAuthFilter` 解析 Token（若有），再根据 URL 匹配到 `IchProjectController.list(...)`。

5. **Controller**  
   - `IchProjectController` 的 `list` 方法接收 `keyword, region, heritageLevel, ichType, page, size`。  
   - 调用 `projectService.list(...)`，得到 `Page<IchProject>`，再封装成 `{ success: true, data: 列表, total, totalPages }` 返回。

6. **Service**  
   - `IchProjectService.list()` 里根据参数拼好条件，调用 `projectRepository.searchByConditions(...)`。

7. **Repository**  
   - `IchProjectRepository` 的 `searchByConditions` 是 JPA 的 `@Query`，生成 SQL 查 MySQL 的 `ich_project` 表，结果映射成 `IchProject` 列表返回。

8. **原路返回**  
   - 数据库结果 → Repository → Service → Controller → HTTP 响应（JSON）→ 前端 axios 收到 `res`。  
   - `Projects.vue` 里把 `res.data` 赋给 `list`，页面用 `v-for` 渲染列表。

答辩时可以说：**“例如非遗项目列表，前端 Projects.vue 调用 getProjectList，请求到后端 IchProjectController，再经 IchProjectService 和 IchProjectRepository 查 MySQL，最后把 JSON 返回给前端渲染。”**

## 八、如何运行（答辩前自己跑一遍）

1. **数据库**  
   - 安装 MySQL，在 `application.properties` 里配置的库中建库（若用 JPA 的 ddl-auto=update，表可自动建）。  
   - 保证库名、用户名、密码与配置一致。

2. **后端**  
   - 用 IDEA 打开项目，找到 `DemoApplication.java`，运行 `main` 方法。  
   - 控制台出现 Spring Boot 启动日志，默认端口 8080。

3. **前端**  
   - 在项目根目录下进入 `frontend`，执行 `npm install`，再 `npm run dev`。  
   - 浏览器打开 Vite 提示的地址（一般是 http://localhost:5173）。  
   - 用户端首页、管理员后台（如 /admin）分别用不同账号登录测试。

答辩时可以说：**“先启 MySQL，再启后端 Spring Boot，最后启前端 npm run dev，浏览器访问 5173 即可。”**
